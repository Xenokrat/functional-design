# Паттерн для Find Matches 2

На текущий момент я уже знаю, что отталкиваясь от ООП первым в голову должен прийти полиморфизм.
Полиморфизм позволит нам создать иерархию классов типа Match, каждый из которых может обрабатывать
соответствующую комбинацию шариков, например Match3, Match4, Match4Cross (пересечения).
Также, если подумать, можно включить в Match стратегию, которая сможет обрабатывать подсчёт очков, и
также может полиморфно "встраиваться" в объект, за счёт общего интерфейса.
